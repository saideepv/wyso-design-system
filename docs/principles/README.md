# Principles

The "why" behind the system. Not yet formally written — this is a seed
list pulled from the first foundations prototype, to be expanded into
real principles (one paragraph each: what it means, what it rules out).

- **Interactive** — the product is a conversation, not a worksheet.
- **Safe space** — no red, no harsh error states; "retry" reads as warm
  (gold), never as failure.
- **Organic** — imperfect blob shapes and gentle tilt over rigid grids and
  perfect circles, except where a shape is functional (e.g. the mic
  button).
- **Game × learning** — celebration and reward moments are first-class UI,
  not an afterthought.
- **Modern** — confident colour and type, not a "kids app" pastel default.
