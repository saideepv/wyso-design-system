# Foundations

The narrative and rationale behind each token layer in `tokens/`. The CSS
lives there; the *why* lives here. One doc per foundation, eventually:

- `color.md` — the "Lemon Mint" palette, why mint is capped at ≤10% of a
  screen, why red/grey/purple are excluded.
- `typography.md` — why Baloo 2 + Atkinson Hyperlegible, Devanagari
  support, the type scale and when to step up/down it.
- `spacing.md` — the 4px base unit, hit-target minimums for a touch-first,
  middle-school audience.
- `shape-and-motion.md` — organic blobs vs. functional circles, gradient
  usage, when tilt is and isn't allowed.

None of these have been written yet — `tokens/README.md` is the closest
thing to a foundations doc for now.
